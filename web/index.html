mcp-sandbox

A public MCP server of test fixtures for QA and evals.

Tools return deliberately distinctive output, so a caller can prove a response came from this server — not from the model's own training/context.

streamable HTTP   Endpoint: /mcp

Tools

ToolParamsReturns
identity none A fixed canary token. Seeing it confirms the MCP server answered.

Add to Claude Code

claude mcp add --transport http mcp-sandbox URL

Add to Claude Desktop

Desktop config is stdio-only, so bridge through mcp-remote (claude_desktop_config.json):

{
  "mcpServers": {
    "mcp-sandbox": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "URL"]
    }
  }
}

Try it

Ask the model:

Call the mcp-sandbox `identity` tool and show me what it returned.
For evals, assert on the raw tool result rather than the model's prose — the token is stable, the wording isn't.
