Why mcp-sandbox?

While building or testing an agent you usually don't want to connect to a production MCP server. You want deterministic tools that always return a known output — so you can tell whether the MCP layer actually worked.

Tools

Endpoint: /mcp  (no trailing slash). Each tool returns a fixed canary — show it verbatim.

identity

params: none

Confirms the server is connected. Returns a fixed ID string.

ReturnsMCP-SANDBOX canary 7Q9X-ZK42-VORTEX-PARSNIP

echo

params: message: str, repeat: int = 1

Sends your text through the server and returns it inside a fixed wrapper, repeated repeat times.

ReturnsMCP-SANDBOX echo 4F7T-ECHO-START >>> your text <<< 4F7T-ECHO-END

auth_check

params: none (reads Authorization header)

Verifies the server saw your Authorization: Bearer <token> header.

ReturnsMCP-SANDBOX auth OK 9B3K-AUTHED-MARIGOLD

Connect in one step

Pick your client. The URL below is filled in live from this page's address.

Claude Code

One command, with the dev Bearer token:

claude mcp add --transport http mcp-sandbox URL \
  --header "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT"

Claude Desktop

Config is stdio-only — bridge via mcp-remote in claude_desktop_config.json:

{
  "mcpServers": {
    "mcp-sandbox": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote", "URL",
        "--header", "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT"
      ]
    }
  }
}

Raw HTTP

Call the MCP endpoint directly with curl:

curl -X POST URL \
  -H "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call",
       "params":{"name":"identity","arguments":{}}}'

⚠️ Endpoint is /mcp with no trailing slash — /mcp/ 307-redirects to /mcp and some clients drop the POST body across the redirect.

Prompt → canary → proof

Ask a connected model to call a tool. The token it relays is what you assert on.

PromptCall the mcp-sandbox identity tool and show what it returned.
↓
ResponseMCP-SANDBOX canary 7Q9X-ZK42-VORTEX-PARSNIP
↓
WhyThe token appears → the call reached this server. No token → the tool was never invoked.
PromptCall the mcp-sandbox auth_check tool and show the result.
↓
ResponseMCP-SANDBOX auth OK 9B3K-AUTHED-MARIGOLD
↓
WhyYour Bearer token reached the server. Without a valid token you get 9B3K-NOAUTH-CINDER instead.

QA & GenAI evaluations

Assert on the raw tool result, not the model's prose — the token is stable, the wording isn't.

In-memory smoke test with the FastMCP client:

import asyncio
from fastmcp import Client

async def main():
    async with Client("URL") as c:
        result = await c.call_tool("identity", {})
        assert "7Q9X-ZK42-VORTEX-PARSNIP" in result.content[0].text

asyncio.run(main())

Authentication

The auth_check tool is gated by a public development Bearer token.

Dev token public · not a production credential

Send it in the Authorization header. It's a shared smoke-test token for integration testing — it is public, intended for testing, and only gates the auth_check tool.

Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT
