mcp-sandbox

Free and open MCP server for developers and GenAI QA evals.

Each tool returns a deliberately distinctive canary token — something a model would never volunteer on its own. Seeing it in a response proves the answer came from this server, not from the model's training or context. Ideal for wiring up MCP clients, smoke-testing connectivity, and asserting on tool calls in evals.

  • Free & open
  • Streamable HTTP
  • Bearer-token auth check
  • Stable canary tokens

Tools

Endpoint: /mcp  (no trailing slash).

identity

params: none

Confirms the server is connected. Returns a fixed ID string — show it verbatim.

MCP-SANDBOX canary 7Q9X-ZK42-VORTEX-PARSNIP

echo

params: message: str, repeat: int = 1

Sends your text through the server and returns it inside a fixed wrapper, repeated repeat times.

MCP-SANDBOX echo 4F7T-ECHO-START >>> your text <<< 4F7T-ECHO-END

auth_check

params: none (reads Authorization header)

Verifies the server saw your Authorization: Bearer <token> header.

MCP-SANDBOX auth OK 9B3K-AUTHED-MARIGOLD

Integrate

Add to Claude Code (one command, with the dev Bearer token):

claude mcp add --transport http mcp-sandbox URL \
  --header "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT"

Add to Claude Desktop — config is stdio-only, so bridge through mcp-remote in claude_desktop_config.json:

{
  "mcpServers": {
    "mcp-sandbox": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote", "URL",
        "--header", "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT"
      ]
    }
  }
}

Raw HTTP — call the MCP endpoint directly:

curl -X POST URL \
  -H "Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call",
       "params":{"name":"identity","arguments":{}}}'

Prompt & expected response

Once connected, ask the model to call a tool. The model relays the canary verbatim — that exact token is what you assert on.

PromptCall the mcp-sandbox identity tool and show me what it returned.
Expected MCP-SANDBOX canary 7Q9X-ZK42-VORTEX-PARSNIP

If the token appears, the call reached this server. If the model answers without it, the tool was never invoked.

PromptCall the mcp-sandbox auth_check tool and show the result.
Expected MCP-SANDBOX auth OK 9B3K-AUTHED-MARIGOLD

Confirms your Bearer token reached the server. Without a valid token you get MCP-SANDBOX auth FAIL 9B3K-NOAUTH-CINDER instead.

Auth — Bearer token

Dev token (public, for integration testing)

Send it in the Authorization header. This is a shared smoke-test token, not a secret — it only gates the auth_check tool.

Authorization: Bearer mcp-sandbox-dev-C7H2-KPMERCHANT

For evals, assert on the raw tool result rather than the model's prose — the token is stable, the wording isn't.
